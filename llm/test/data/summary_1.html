파일 시스템에 대한 요약

디렉토리

  • 정의: 디렉토리는 파일에 관한 정보를 포함하고 있는 특별한 파일이며, 파일 이름과 파일 자체 사이의 매핑을 제공합니다.

파일 시스템의 필수 요구사항

  • 정보는 매우 큰 양을 저장할 수 있어야 함
  • 프로세스 종료 후에도 정보가 유지되어야 함
  • 다수 프로세스가 동시에 정보에 접근 가능해야 함

파일과 파일 시스템

  • 파일: 저장 장치에서 정보를 저장하는 기본 단위, 바이트 배열, 사용자에 의해 단일 엔티티로 처리되며, 고유한 경로 이름과 접근 제어를 가짐
  • 파일 시스템: 파일 사용에 대해 사용자와 어플리케이션에 서비스를 제공하는 시스템 소프트웨어 집합

파일 확장자와 파일 타입

  • 전형적인 파일 확장자 및 파일 타입 예시 제공

파일 속성과 접근 권한

  • 파일에는 여러 속성들이 있으며, 이를 통해 파일에 접근하고 관리할 수 있음

파일 시스템 접근

  • 순차 접근: 파일의 맨 앞에서부터 읽기/쓰기
  • 직접/랜덤 접근: 파일을 열고 지정된 위치에서 데이터를 읽거나 쓸 수 있는 방식

파일 공유

  • 멀티 프로세스가 동시에 파일을 열 수 있는지 여부 및 파일 변경사항이 다른 프로세스에 언제 영향을 미치는지와 관련된 규칙

파일 작업과 시스템 호출

  • 파일 생성, 삭제, 이름 변경, 속성 읽기/설정 등의 작업과 관련된 일반적인 시스템 호출 목록

파일 시스템의 구현과 관리

  • 디렉터리 구조: 단일 레벨과 계층형 디렉터리 시스템
  • 패스 이름: 절대 경로와 상대 경로

디렉터리 관리

  • 디렉터리 작업에 대한 시스템 호출 목록

파일 할당 방법

  • 연속 할당: 파일 생성 시 하나의 연속적인 블록 세트가 할당됨
  • 체인 할당: 각 블록이 체인의 다음 블록을 가리키는 포인터를 포함
  • 색인 할당: 파일의 각 부분을 가리키는 인덱스가 유지됨

파일 시스템 최적화

  • 파일 시스템 레이아웃과 관련된 최적화 방법에 대한 설명
